GIT操作最基本流程

​ 创建工作目录workspace 对工作目录进行修改

​ git add ./

​ git hash-objiect -w 文件名

​ git update-index …

​ git commit -m “注释内容”

​ git write-tree

​ git commit-tree

GIT高层命令

​ git init 初始化仓库

​ git status 检查文件当前状态（已跟踪=>已修改，已暂存，已提 交，未跟踪)

​ git diff 查看当前哪些修改还没有暂存

​ git diff –staged 查看哪些修改已经被暂存还没有被提交

​ git log –online 查看提交记录

​ git rm 文件路径 删除文件再将修改添加到暂存区

​ git mv 源文件名 新文件名 修改文件再将修改添加到暂存区

​ git add ./ 将修改添加到暂存区

    git  commit -m "注释内容"             将暂存区提交到版本库

​ git commit -a - m 跳过暂存区

​ git commit vim

​

分支

​ 注意：在切换分支的时候要确保当前分支是干净的

​ 允许切换分支：

​ 分支上所有内容处于已提交状态

​ （避免）分支上的内容是初始化创建 处于未跟踪状态

​ （避免）分支上的内容是初始化创建 第一次处于已暂存状态

​ 不允许切换分支：

​ 分支上所有内容处于已修改状态 或 第二次以后的已暂存状态

​ git branch 显示分支列表

​ git branch 分支名 创建分支

​ git checkout 分支名 切换分支

​ git checkout -b 新分支名 切换分支同时切换到刚新建的分支上

​ git branch -D 分支名 强制删除分支

​ git branch -d 分支名 删除合并分支

​ git config –global alias.别名 status 命令行配置别名

​ git merge 分支名 合并分支（快进合并）（典型合并，有机会产生冲突，解决冲突，打开冲突文件 进行修改）

​ git branch –merged 查看合并到当分支的分支

一但出现在这个列表中，就应该删除

​ git branch –no–merged 查看没有合并到当分支的分支

一但出现在这个列表中，应该查案是否需要合并

存储

​ 1. 暂存

​ git stash 会将当前分支上的工作堆到一个栈中、

​ 分支切换，进行其他工作，却回原分支

​ 2. 查看暂存记录

​ git stash list

​ 3. 取回暂存代码

​ git stash apply 将栈中的工作内容还原，但不让任何内容出栈

​ git stash drop 取出栈的工作内容，就应该将其删除

​ git stash pop git stash apply git stash drop

后悔药

​ 工作区：还未git add

​ 如何撤回自己在工作目录中的修改： git restore filename git checkout filename

​ 暂存区：已git add

​ 如何撤回自己在工作目录中的修改： git reset HEAD filename

​ 版本库：git commit

​ 如何撤回自己在工作目录中的修改： git rest —soft HEAD~

reset 三部曲

​ git rest —soft HEAD~ 工作区和暂存区不变 只动HEAD（带着分支一 起移动）

​ git rest [–mixed] HEAD~ 动HEAD（带着分支一起移动）动了暂存区

​ 这是默认情况，直接写 git rest HEAD~ 就会默认加上 —mixer）

​ git rest —hard HEAD~ 动HEAD（带着分支一起移动） 动了暂存区 动了工作区

git branch recover-branch HASH

创建一个新的分支指向这个提交来恢复它

远程协作的基本流程

​ 第一步: 创建一个空的远程仓库

​ 第二步： 创建一个待推送的本地仓库

​ 第三步： 为远程仓库配别名 git remote -v (查看远程别名)

​ git remote add 别名 远程地址 (创建远程仓库别名)

​ 第四步： 在本地仓库中初始化代码 提交代码

​ 第五步： 推送 git push 别名 想要推送的分支

​ 第六步： 邀请成员

​ 第七步： 成员clone远程仓库

​ 第八步： 成员做出修改

​ 第九步： 成员推送自己的修改

​ 第十步： 项目拉取成员的修改
